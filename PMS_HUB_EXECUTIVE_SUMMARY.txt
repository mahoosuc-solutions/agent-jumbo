OVERVIEW
========
The PMS Hub is a multi-provider property management system integration layer
supporting Hostaway, Lodgify, Hostify, and iCal calendars. It provides unified
API access to multiple PMS platforms with event-driven webhook processing and
synchronization with the PropertyManager system.

KEY FINDINGS
============

1. AUTHENTICATION METHODS
   - Hostaway: OAuth 2.0 Bearer Token (no token refresh mechanism)
   - Lodgify: API Key Bearer Token + HMAC-SHA256 webhook verification
   - Hostify: API Key Bearer Token (no webhook verification)
   - iCal: HTTP GET (public URLs only, no authentication)

2. WEBHOOK SECURITY
   ✅ COMPLETE: Lodgify HMAC-SHA256 signature verification (Line 513-539)
   ❌ MISSING: Hostaway signature verification (returns True always)
   ❌ MISSING: Hostify signature verification (returns True always)
   ⚠️  ISSUE: Unsigned webhooks accepted if no signature provided

3. CREDENTIAL STORAGE & SECURITY
   ⚠️  CRITICAL: Plaintext JSON storage at ~/.pms_hub/providers.json
   ✅ GOOD: API responses don't expose credentials
   ❌ BAD: File-level encryption not implemented
   ❌ BAD: No credential rotation mechanism

4. SYSTEM INTEGRATION ARCHITECTURE
   - Provider Registry: Configuration management (JSON-based)
   - Event Bus: Event emission and SQLite storage
   - Sync Service: PMS → PropertyManager synchronization
   - API Endpoints: Webhook receiver, settings get/set
   - Tool Interface: Agent-zero integration

5. DATA FLOW
   Webhook Reception:
   ├─ Webhook received at /api/pms/webhook/{provider}/{provider_id}
   ├─ Signature verified (Lodgify only)
   ├─ Event emitted to EventBus
   ├─ Event stored in ~/.pms_hub/events.db
   └─ ❌ No handlers registered - events not processed

   Tool-Based Queries:
   ├─ Agent-zero calls pms_hub.execute(action, ...)
   ├─ Registry loads provider from ~/.pms_hub/providers.json
   ├─ Provider instance authenticated and cached
   ├─ API calls made with Bearer token authentication
   └─ Canonical models returned

6. CRITICAL GAPS & ISSUES

   Gap 1: Webhook Event Handlers Not Connected (SEVERITY: HIGH)
   - Events emitted but no subscribers registered
   - Sync service never automatically triggered by webhooks
   - Manual sync required (action="sync_reservations")
   - FIX: Subscribe to "pms.webhook.*" events in sync_service

   Gap 2: Hostaway Webhook Verification Missing (SEVERITY: HIGH)
   - verify_webhook() returns True without validation
   - No signature algorithm documented
   - Risk: Webhook spoofing possible
   - FIX: Research Hostaway signature headers, implement verification

   Gap 3: Hostify Webhook Verification Missing (SEVERITY: HIGH)
   - verify_webhook() returns True without validation
   - No signature support documented
   - Risk: Webhook spoofing possible
   - FIX: Research Hostify signature headers, implement verification

   Gap 4: Token Refresh Not Implemented (SEVERITY: MEDIUM)
   - Hostaway OAuth2 tokens expire
   - No refresh token handling
   - Failed API calls when tokens expire
   - FIX: Implement refresh token storage and rotation

   Gap 5: One-Way Synchronization Only (SEVERITY: MEDIUM)
   - PropertyManager → PMS sync not implemented
   - Cannot update PMS when lease changes
   - FIX: Implement reverse sync logic

   Gap 6: No Error Handling/Retry Logic (SEVERITY: MEDIUM)
   - Minimal error handling in API calls
   - No rate limit detection (HTTP 429)
   - No network timeout retry
   - FIX: Add exponential backoff retry mechanism

   Gap 7: Plaintext Credential Storage (SEVERITY: HIGH)
   - API keys stored unencrypted in JSON
   - File accessible to any process running as user
   - FIX: Encrypt credentials using cryptography library

   Gap 8: iCal Feed Credentials (SEVERITY: MEDIUM)
   - No support for authenticated iCal feeds
   - Only public URLs supported
   - FIX: Add HTTP Basic Auth, Bearer token support

WEBHOOK FLOW ARCHITECTURE
=========================

Reception:
  POST /api/pms/webhook/{provider}/{provider_id}
    ↓
Validation:
  ├─ Provider exists and enabled
  ├─ Signature verified (provider-specific)
  └─ Return 401 if invalid

Event Emission:
  event_bus.emit("pms.webhook.{provider}.{event_type}", {
      "provider": "...",
      "provider_id": "...",
      "payload": {...},
      "webhook_timestamp": "..."
  })
    ↓
Event Storage:
  SQLite INSERT INTO events (type, payload, event_hash, created_at)
    ↓
Event Distribution:
  For each handler in _handlers.get(event_type, []):
      await handler(event)
    ↓
⚠️  ISSUE: No handlers registered
    - Events stored but never processed
    - Sync service never triggered

    ├─ Parse ProviderConfig objects
    └─ Cache in memory

EVENT BUS ARCHITECTURE
======================

Storage:
  Location: ~/.pms_hub/events.db (SQLite)
  Schema:
    CREATE TABLE events (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      type TEXT NOT NULL,
      payload TEXT NOT NULL,
      event_hash TEXT NOT NULL,
      created_at TEXT NOT NULL
    )

Operations:
  emit(event_type, payload):
    1. Hash event
    2. INSERT INTO events
    3. Execute registered handlers
    4. Return event dict

  subscribe(event_type, handler):
    1. Add handler to _handlers[event_type]

  Event types: "pms.webhook.{provider}.{event}"

Audit:
  ├─ Event hash verification via audit.py:hash_event()
  ├─ Immutable audit trail
  └─ Complete event history available

API ENDPOINT SUMMARY
====================

POST /api/pms/webhook/{provider}/{provider_id}
  Parameters:
    - provider: hostaway|lodgify|hostify|ical
    - provider_id: registered provider ID
    - signature: webhook signature (optional)
    - payload: event payload
    - timestamp: webhook timestamp

  Response:
    ✅ 200: { status: "success", event_id }
    ❌ 401: { status: "error", message: "Invalid webhook signature" }
    ❌ 404: { status: "error", message: "Provider not found" }
    ❌ 400: { status: "error", message: "Missing parameters" }

GET /api/pms/settings
  Parameters:
    - provider_id: optional, specific provider

  Response:
    ✅ All providers: { providers: [...], total, enabled_count }
    ✅ Single provider: { provider: {...} }
    ⚠️  Credentials NOT exposed (only has_credentials flag)

POST /api/pms/settings
  Actions:
    - add: { action, provider_id, provider_type, name, credentials }
    - update: { action, provider_id, credentials, options }
    - remove: { action, provider_id }
    - enable: { action, provider_id }
    - disable: { action, provider_id }

  Response:
    ✅ 200: { status: "success", message, provider_id }
    ❌ 400: { status: "error", message, valid_actions }
    ❌ 500: { status: "error", message }
